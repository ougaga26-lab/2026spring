# 交接說明：名古屋・伊勢志摩・吉卜力公園 行程網頁

## 專案目標
把一趟9天8夜的日本行程（名古屋 → 賢島 → 伊勢志摩 → 知多半島離島 → 吉卜力公園）做成一個可以發布在 GitHub Pages 的靜態網頁，手機瀏覽為主，內容包含完整逐日行程 + 互動地圖。

## 目前狀態
內容與網頁本體已完成，**尚未部署到 GitHub Pages**。這次交接主要是要請 Claude Code 協助完成部署，以及後續如果有行程異動時的維護。

## 檔案結構
```
/（專案根目錄，四個檔案要放在同一層）
├─ index.html   網頁骨架，引入 Google Fonts、Leaflet 地圖套件、style.css、data.js、app.js
├─ style.css    外觀樣式（顏色、字體、版面），設計概念是「集章旅行手帳」
├─ data.js      行程內容資料（唯一預期會常常被修改的檔案）
└─ app.js       渲染邏輯：讀 data.js 畫出手帳頁與地圖，一般不需要更動
```

## 技術棧
- 純靜態網頁，**沒有任何框架、沒有 build step、沒有 npm 套件**
- 地圖：Leaflet.js + OpenStreetMap 圖磚（透過 CDN 引入，**不需要任何 API 金鑰**）
- 字體：Google Fonts（Noto Serif TC / Noto Sans TC / JetBrains Mono）
- 手機優先的 RWD，最大寬度 480px 置中

## data.js 的資料結構（之後行程異動時修改這裡）
```js
const TRIP = {
  title, subtitle, dateRange, stats: [...],
  days: [
    {
      id: "day4",                 // 錨點用，改動時建議保持穩定
      seal: "四",                 // 印章上顯示的字（可用中文數字/符號）
      dateLabel: "9/23（三）",
      title: "伊勢 → 渡輪 → 日間賀島",
      timeline: [                 // 當天的時間軸項目，依序顯示
        {
          kind: "ferry",          // transport | food | sight | sunrise | ferry | onsen | note
          time: "10:40",
          title: "搭伊勢灣渡輪",
          note: "船程約60分，跨越伊勢灣",
          lat: 34.480432, lng: 136.846764   // 有經緯度才會畫在地圖上
        },
        ...
      ],
      hotel: { name, note, lat, lng },  // 沒有住宿當天填 null（如最後一天）
      alert: "提醒文字"                  // 選填，會用紅色提示框顯示
    },
    ...
  ]
};
```
`kind` 決定時間軸上的顏色跟 icon（對照表在 `app.js` 開頭的 `ICONS` 物件），新增 kind 類型要同步在 `app.js` 和 `style.css`（`.tl-item[data-kind="..."]`）加對應樣式。
